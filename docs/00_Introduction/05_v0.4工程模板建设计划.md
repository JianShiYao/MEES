## 3. 建设原则

1. **一个工作产品、一个最终责任角色**：模板不得重新制造总控过程与专业过程的双重所有权。
2. **内容与治理分离**：系统/软件负责人拥有专业规格内容，需求、架构和配置治理角色维护状态、追溯、评审和基线记录。
3. **模板与实例分离**：`templates/` 保存规范模板，`docs/13_Templates/` 保存目录和使用说明，`docs/15_Case_Study/` 保存脱敏实例与复盘。
4. **证据状态诚实**：空模板是 `D` 类定义证据；模拟填写是 `S`；真实项目记录经确认后才是 `P`；独立评价记录才是 `I`。
5. **先核心闭环，再专项深化**：先保证项目、需求、架构、验证、配置和发布可运行，再补齐功能安全、网络安全、供方和组织改进资产。
6. **字段可验证**：标识、状态、版本、所有者、评审、批准、适用性、追溯和归档字段采用稳定名称，便于 v0.5 自动检查。
7. **不复制标准原文**：模板只表达原创工程字段、检查问题和使用指导。

## 4. 模板资产结构

### 4.1 存放规则

| 位置 | 内容 | 发布要求 |
|---|---|---|
| `templates/00_Governance/` | 项目、裁剪、元数据、QA、能力和改进模板 | 可直接复制，不含项目实例 |
| `templates/10_Engineering/` | 需求、架构、接口、验证和追溯模板 | 字段与 G2–G5 对齐 |
| `templates/20_Safety/` | 相关项、HARA、安全概念、确认措施、安全分析和 `EXT-HW` | 不预设 ASIL 或符合性结论 |
| `templates/30_Cybersecurity/` | 安全范围、TARA、漏洞、更新、SBOM 和安全指南 | 不预设 SL 或认证结论 |
| `templates/40_Release_Quality/` | 发布、供方、指标、CAPA 和归档模板 | 与 G6、质量及配置控制对齐 |
| `docs/13_Templates/` | 模板目录、选择指南、字段词典和复制说明 | 纳入 MkDocs 导航 |

### 4.2 每项模板的最小组成

- 模板标识、模板版本、适用工作产品、最终责任角色和来源差距。
- 项目标识、工作产品标识、状态、版本、作者/提供者、评审者、批准者和日期。
- 目的、适用范围、输入、填写规则、正文或台账字段、评审准则、裁剪规则和归档要求。
- 上游、下游、验证、问题/变更和基线/发布追溯字段。
- `D/S/P/I` 证据状态声明以及“不得据此声明认证或能力等级”的适用边界。
- 至少一个最小填写示例或 BMS 试点引用，但规范模板本身不得混入项目事实。

统一占位符使用 `[填写：字段说明]`；`TBD` 必须同时给出责任人和关闭时间，不能作为批准基线中的永久值。

## 9. 进入 v0.5 的门禁

只有满足以下条件，才把模板字段作为自动化与度量的稳定输入：

1. V4-G5 通过，v0.4.0 模板基线和实例边界已冻结。
2. 模板标识、工作产品标识、状态、所有者、版本和追溯字段命名稳定。
3. 至少一套 BMS 实例证明字段可以被真实填写，不只在空模板中存在。
4. 已形成可自动检查的规则清单，包括目录、必填字段、占位符、追溯和版本一致性。
5. 指标与质量目标字典给出数据源、频率、责任和行动阈值。

## 11. 实施顺序

1. 先完成 WP0 和一个工作产品元数据样板，冻结 V4-G1。
2. 并行建设 WP1/WP2，但以 `TPL-001`、`002`、`003`、`004`、`005`、`006` 为首批可运行集合。
3. 使用 MK8 状态机候选增量对首批模板做一次小闭环，问题立即反馈模板，不等待全部专项资产完成。
4. 建设 WP3/WP4，并按 BMS 的真实适用性决定 HARA、TARA、`EXT-HW`、SBOM 和安全更新实例深度。
5. 完成至少 9 项必须模板实例化、复制测试和人工评审后冻结 V4-G4 候选。
6. 关闭严重和主要问题，形成 V4-G5 发布决定和 v0.5 自动化输入。

## 12. 主要风险

| 风险 | 控制措施 |
|---|---|
| 模板数量多但无法实际填写 | 每个工作包尽早进入 MK8 小闭环，V4-G4 检查复制和填写结果 |
| 同一字段在多个模板重复维护 | 定义字段词典和主记录位置，其他模板只引用标识 |
| 功能安全/网络安全模板被误解为符合性证明 | 强制适用性、证据状态和非认证声明，领域负责人评审 |
| 模板过度通用或过度绑定 BMS | 规范模板保持产品无关，BMS 内容只进入案例实例 |
| 角色权威表缺少领域负责人 | V4-G1 前修订核心角色表并关闭 `FS-001/CS-001` 的角色部分 |
| v0.5 自动化过早绑定不稳定字段 | V4-G5 前只记录自动化需求，不承诺稳定接口 |
