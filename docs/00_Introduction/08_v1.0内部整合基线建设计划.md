## 1. 目的

把 v0.1–v0.6 已建的过程、映射、模板、检查表、Agent、自动化和案例整合为**一份干净、可评审、可裁剪、内部一致**的 v1.0 基线，消除跨版本累积的债务与矛盾，形成稳定的组织过程资产**参考底本**。

## 2. 定位与边界（务必先读）

**v1.0 是内部整合基线**，不是组织级落地或认证。沿用一贯边界：

- 证据仍为工具 `P`（工具可执行）+ 数据 `S`（MK8/ESS 模拟）；**无真实项目 `P` 或独立评价 `I`**。
- 签署仍为**模拟审计**（SIM-QA-001）+ JianShi 人工，**非真实独立审计、第三方评估、标准符合性、认证或能力等级**。
- Agent 只读、不批准门禁；产品保持 `No-Go`。

**升级为"真实组织级 1.0"的退出判据（本版明确不在范围内）**：① 由真实且具独立性的评审人复核签署（替代 SIM-QA-001）；② 至少一个真实项目按过程执行并产出受控 `P`/`I` 证据；③ 对口目标领域标准落实（储能：IEC 61508 / UL 1973 / UL 9540A / IEC 62619 / IEC 62443 / IEEE 1547）。达成这三项前，不得对外声称组织级基线。

## 3. 输入基线

- Git 标签 `v0.6.0`（候选 `e99be73`，CI `mees-checks` 绿）。
- v0.1–v0.6 全部受控资产：11 过程 + 标准映射（ASPICE/33020/26262/62443/9001 + 储能安全轨）+ 18 模板 + `TPL-V05-*` + 检查表 + 六类 Agent + Obsidian 适配器 + 自动化脚本（`check_mees`/追溯/指标/Dashboard/`run_all_checks`）+ 案例走查。
- 各版本评审记录、差距台账与遗留项（`GAP-V03-*`、`ESS-*`、`M6-*` 等）。

## 4. 建设目标

1. **一致性**：全语料版本、状态、证据标签、完成度与门禁结论互不矛盾。
2. **编号债**：解决 `LIF` 前缀被泛化（映射/台账挤入 LIF）的历史债，使编号规范与实际一致。
3. **权威边界与证据声明**：建立**单一权威**文档，集中说明证据分层、模拟审计边界、产品 No-Go、领域定位与"真实组织级 1.0 退出判据"。
4. **统一台账与索引**：合并跨版本差距/遗留项为一份台账；为多份评审记录/走查建立导航索引。
5. **全语料完整性**：`run_all_checks` 全绿、无孤立页/断链/编号冲突，Dashboard 作为 1.0 健康快照。
6. **可复现发布**：冻结候选、干净导出复现、GitHub CI 绿、模拟审计签署、`v1.0.0` 标签。

## 5. 非目标

- 不新增能力域内容、不新建标准映射、不做真实项目部署。
- 不退出模拟审计、不把 `S` 升级为 `P/I`。
- 不重写既有已发布版本的历史标签与评审记录（保持标识稳定，仅整合与索引）。
