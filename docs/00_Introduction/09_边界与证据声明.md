## 1. 目的与地位

本页是 MEES 关于**证据等级、评审性质、产品状态与能力边界**的**单一权威声明**。全库其他文档对这些概念的表述以本页为准；如有冲突，以本页为准。本页只做定义与边界声明，不改变任何已发布记录的结论。

## 2. 证据状态模型（D / S / P / I）

| 状态 | 名称 | 判定边界 |
|---|---|---|
| D | 文档定义证据 | 过程、规则、模板或映射已经定义，但没有项目执行记录 |
| S | 模拟演练证据 | 桌面走查或模拟角色评审已经完成，明确不是真实项目部署 |
| P | 项目实施证据 | 真实项目按过程执行并保留受控记录、工具数据和结果 |
| I | 独立评价证据 | 具备独立性的真实评审、审核或评估已经形成可审计结论 |

证据状态可组合（如 `D+S`）。**不得以高状态替代低状态的内容完整性，也不得把 `D`/`S` 表述为 `P`/`I`**。工具可执行性与数据来源分别标注（例如"工具 `P` + 数据 `S`"）。

## 3. 评审与签署的性质

- **模拟审计**：用于验证评审、整改与发布**流程和记录结构**的演练，**不构成真实独立评价或认证**。
- **SIM-QA-001**：模拟独立评审身份，**为虚构、非真实人员**，仅用于走通独立评审流程；其签署不等于真实独立审计。
- **JianShi 人工签署**：仓库内部人工决定，非真实独立评估。
- 各版本门禁签署（V*-G*）均为**仓库内部决定**，不构成标准符合性、能力等级、认证或组织级结论。

## 4. 产品与能力边界

- **产品状态：`No-Go`。** MEES 当前为研发工程知识库与方法基线，未发布任何受控产品；所有版本发布（`vX.Y.0`）指**知识库/方法基线**发布，不是产品发布。
- **能力等级**：ISO/IEC 33020 框架下基于过程属性评级的结果，**不由文档数量或本库存在推定**；本库未形成真实能力等级结论。
- **标准符合性**：本库为对标准的**原创工程解读与映射**，不含标准原文，不构成符合性声明或认证。
- **Agent 边界**：六类 AI Agent 为**只读辅助**，输出均需人工复核，**不批准任何门禁**，不改变产品 `No-Go`。
- **Obsidian 边界**：O0–O2 只读与 O3 **受控本地文件写入**已试点（V6-G3/G4 已签，写入默认关闭）；**经 REST 写入 Obsidian vault 未实现，属后续能力**。

## 5. 版本定位边界

- **v1.0 为"内部整合基线"**，指将 v0.1–v0.6 整合为干净、可评审、可裁剪的**内部**基线，**不是真实组织级 1.0 落地**。
- 真实组织级 1.0 的退出判据（真实独立评审 `I`、真实项目 `P` 证据、领域标准符合）见 [v1.0 内部整合基线建设计划](08_v1.0内部整合基线建设计划.md)。

## 6. 术语口径（与边界相关）

| 术语 | 统一含义 |
|---|---|
| 覆盖 | 在指定范围和证据状态下存在可说明的承接关系，**不等于符合性结论** |
| 适用性 | 基于产品、项目、法规、合同和裁剪条件判断某关注点是否进入范围 |
| 模拟审计 | 用于验证流程和记录结构的演练，不构成真实独立评价或认证 |
| 能力等级 | ISO/IEC 33020 框架下基于过程属性评级的结果，不由文档数量推定 |
