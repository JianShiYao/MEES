# MEES 首页

欢迎进入 **MEES — 现代嵌入式研发工程体系**。

## 使用路径

- 管理者：先阅读 [MEES 总览](00_Introduction/00_MEES总览.md) 和 [建设路线图](00_Introduction/01_建设路线图.md)。
- 项目经理：阅读 [项目管理过程](01_Main_Process/01_项目管理过程.md)、风险、质量策划与度量。
- 系统工程师：阅读 [需求管理过程](01_Main_Process/02_需求管理过程.md)、[架构设计过程](01_Main_Process/03_架构设计过程.md)、系统验证。
- 软件工程师：阅读软件需求、架构、详细设计、编码与单元验证。
- 测试工程师：阅读 [验证确认过程](01_Main_Process/04_验证确认过程.md)、集成测试、系统测试、确认测试与自动化测试。
- 质量人员：阅读 [ASPICE + ISO/IEC 33020 过程映射表](11_Process_Management/ASPICE_ISO_IEC_33020过程映射表.md)、[ISO/IEC 33020 中文工程解读](11_Process_Management/ISO_IEC_33020中文工程解读.md)、审核、[配置管理过程](01_Main_Process/06_配置管理过程.md) 和度量。
- 发布负责人：阅读 [发布管理过程](01_Main_Process/05_发布管理过程.md)、[配置管理过程](01_Main_Process/06_配置管理过程.md) 和发布检查表。

## 当前建设重点

1. 建立统一目录和写作规范。
2. 完成 Agile + Mini V 全景架构。
3. 完成项目管理、需求管理、架构设计、验证确认、发布管理和配置管理首批主流程文档。
4. 建立 ASPICE + ISO/IEC 33020 过程映射。
5. 扩展模板、检查表和 Agent 目录。
