# MEES 指标字典（v0.5 WP3）

> 文档编号：MEES-LIF-013
> 版本：v0.5.1-dev
> 状态：评审中（v0.5.1 收口候选）
> 所有者：质量负责人
> 最后更新：2026-07-15

## 1. 目的

冻结 v0.5 首批质量指标的口径：标识、目的、公式、数据源、粒度、责任角色、采集频率、阈值、异常处置、证据状态与限制。指标由 `scripts/collect_metrics.py` 采集，输出 `build/metrics/metrics.json`。

指标用于**过程扫描与问题定位**，不单独用于绩效考核，也不构成能力等级或符合性结论。

## 2. 通用规则

- **粒度**：默认仓库/候选提交级；追溯类指标可下钻到对象。
- **采集频率**：每次候选提交与 PR；发布前必采。
- **N/A 规则（反虚假绿灯）**：分母为 0、证据仅 `S`、或数据源未接入时，指标输出 `N/A（无充分证据）`，**绝不显示误导性的 0% 或 100%**。`MET-VNV-003` 在无已执行测试时即为此类。
- **证据分层**：`D` 文档定义 / `S` 模拟走查 / `P` 真实项目 / `I` 独立评价。指标继承其数据源的证据等级，自动化不得升级证据。
- **阈值语义**：阈值触发**审阅与处置**，不自动判定通过/失败或批准门禁。

## 3. 指标定义

| 标识 | 指标 | 目的 | 公式 | 数据源 | 责任角色 | 阈值（审阅触发） | 证据 |
|---|---|---|---|---|---|---|---|
| `MET-DOC-001` | 文档检查通过率 | 监控链接/编号/构建/导航健康度 | 无 error 诊断文件 / 受控文件 | `check_mees` report.json | 配置管理员 | < 100% | D |
| `MET-TPL-001` | 模板实例完整率 | 识别必填字段/占位符缺口 | 无未处置 `[填写：]` 实例 / 模板实例 | 模板实例扫描 | < 100% | S |
| `MET-TRC-001` | 双向追溯完整率 | 识别孤立对象 | 非孤立对象 / 全部对象 | traceability.json | 系统负责人 | < 90% | S |
| `MET-VNV-001` | 需求验证覆盖率 | 识别无验证技术需求 | 有验证关系技术需求 / 技术需求 | traceability.json | 测试负责人 | < 100% | S |
| `MET-VNV-002` | 测试执行率 | 区分计划/已执行 | 已执行测试 / 计划测试 | traceability（测试状态） | 测试负责人 | 高亮 0 执行 | S |
| `MET-VNV-003` | 测试通过率 | 仅对已执行测试 | 通过测试 / 已执行测试 | 测试执行记录 | 分母 0 → N/A | S |
| `MET-ISS-001` | 开放问题严重度分布 | 支撑升级与发布风险 | 按严重度计数开放问题 | 问题台账（待接入） | 严重 > 0 | - |
| `MET-REL-001` | 发布门禁就绪度 | 汇总 G0–G6 必须证据 | 就绪门禁 / 适用门禁 | 门禁证据（待接入） | < 100% 不发布 | - |
| `MET-QA-001` | 不符合项逾期率 | 支撑 QA 与 CAPA 升级 | 逾期 NC / 开放 NC | QA 记录（待接入） | > 0 | - |
| `MET-SUP-001` | SBOM/供应方证据完整率 | 识别组件/许可/外部硬件缺口 | 有证据组件 / 组件 | SBOM 实例（待接入） | < 100% | - |

## 4. 当前采集结果（MK8 模拟，候选提交级）

由 `collect_metrics.py` 计算；`S` 数据来自 MK8 试点，`na` 为无充分证据。

| 标识 | 当前值 | 说明 |
|---|---|---|
| `MET-DOC-001` | 100% | 受控 Markdown 全部无 error 诊断 |
| `MET-TPL-001` | 100% | 9 项试点实例无未处置占位符 |
| `MET-TRC-001` | ~76% | 存在孤立/未定义对象，追溯尚不完整 |
| `MET-VNV-001` | 100% | 技术需求均有验证关系 |
| `MET-VNV-002` | 0% | 6 个用例计划未执行（高亮） |
| `MET-VNV-003` | **N/A** | 无已执行测试，分母为 0 → 不显示 0%/100% |
| `MET-ISS-001` / `REL-001` / `QA-001` / `SUP-001` | **N/A** | 口径已定义，结构化数据源未接入 |

## 5. 限制

- MK8 数据为模拟 `S`；所有百分比不代表真实产品质量或过程能力。
- 四项 `na` 指标的采集依赖后续版本接入问题台账、门禁证据、QA 记录与结构化 SBOM。
- 指标不替代需求、安全、测试、质量与发布角色的专业判断与签署。
