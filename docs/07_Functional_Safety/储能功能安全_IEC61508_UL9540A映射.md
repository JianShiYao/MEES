# 储能功能安全与领域安全映射（IEC 61508 / UL 9540A 轨）

> 文档编号：MEES-MAP-001
> 版本：v0.5.1-dev
> 状态：评审中（v0.5.1 收口候选）
> 所有者：功能安全负责人

> 本文是 MEES 面向固定式储能/电池系统的原创工程接口映射，基于 IEC 61508、UL 1973、UL 9540A、IEC 62619 与 IEC 62933 系列，不是标准译文、符合性声明或安全认证结论。

## 1. 目的与范围

建立**固定式储能/电池系统**功能安全与领域安全生命周期，与 MEES 产品、项目、系统、软件、测试、配置、变更和发布过程之间的责任、工作产品、门禁与证据接口。本轨与汽车 [ISO 26262 过程映射](ISO_26262过程映射.md) **并列**：ISO 26262 用 ASIL/HARA，本轨用 IEC 61508 **SIL** 与热失控为核心的领域危害分析。

本轨关闭 [`GAP-V03-FS-009`](../11_Process_Management/v0.3差距与行动项.md) 的结构部分。映射为 `D` 级文档定义，不代表真实项目、SIL 决策、型式试验或安全案例证据，也不声明任何标准符合性。

## 3. 适用性与责任边界

- **激活条件**：项目属于固定式储能/电池系统范围，或合同/法规要求采用等效方法时启用本轨；汽车项目改用 ISO 26262 轨。
- 安全完整性以 **SIL（IEC 61508）** 表达，不使用 ASIL；危害分析以**热失控与电气/化学危害**为核心，不套用道路车辆运行场景。
- 领域安全的实质大量位于**硬件/电化学**层：电芯、电池包、热管理与结构按 `EXT-HW` 外部受控对象承接，但其安全证据（型式试验、热失控数据、供方安全声明）必须保留供方、接口、验收准则与变更通知。
- 无真实 SIL 决策、型式试验、热失控测试或安全案例证据前，不得声明功能安全或产品安全符合性。

## 6. 与 ISO 26262 轨的关系

| 维度 | ISO 26262 轨（汽车） | 本轨（固定式储能） |
|---|---|---|
| 完整性度量 | ASIL | **SIL（IEC 61508）** |
| 危害分析 | HARA、运行场景 | **热失控/电气/化学危害（UL 9540A 为核心测试）** |
| 电池/电芯安全 | 相关项内 | **UL 1973 / IEC 62619（大量 `EXT-HW`）** |
| 系统层安全 | 相关项定义 | **IEC 62933-5-2、现场安装（NFPA 855）、并网（IEEE 1547）** |
| 激活 | 道路车辆项目 | 固定式储能/电池项目 |

两轨互斥激活，共用 MEES 过程、工作产品唯一责任与 G0–G6 门禁；同一项目按产品领域选择其一，不得混用完整性度量。
