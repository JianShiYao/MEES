# v0.4 模板目录与使用规则

> 文档编号：MEES-LIF-011
> 版本：v0.4.0-dev
> 状态：评审中
> 所有者：配置管理员
> 最后更新：2026-07-14

## 1. 目的

定义 v0.4 工程模板的目录、元数据、占位符、复制、裁剪、追溯、评审、基线和归档规则，使 18 项模板可以被项目一致使用并被自动检查。

## 2. 三层资产边界

| 层 | 位置 | 作用 | 证据状态 |
|---|---|---|---|
| 规范模板 | `templates/` | 产品无关、可复制的受控模板 | `D` |
| 使用说明 | `docs/13_Templates/` | 目录、字段、选择和维护规则 | `D` |
| 项目实例 | `examples/` 或受控项目库 | 填写后的项目记录 | 模拟为 `S`，真实确认后为 `P` |

规范模板不得包含客户、产品或个人敏感信息；项目实例不得回写到规范模板。

## 3. 通用元数据

每项规范模板必须包含以下字段：

| 字段 | 规则 |
|---|---|
| 模板标识 | 使用 `TPL-V04-NNN`，在 v0.4 目录内唯一 |
| 模板版本 | 候选使用 `v0.4.0-dev`，批准发布后使用 `v0.4.0` |
| 优先级 | 仅为 `P0` 或 `P1`，与 v0.3 输入一致 |
| 状态 | 草稿、评审中、已批准或已废止 |
| 适用工作产品 | 说明模板生成的唯一工作产品或用途 |
| 最终责任角色 | 必须使用核心过程总览权威名称 |
| 来源差距 | 至少关联一个 `GAP-V03-*` 或明确内部工程需要 |
| 证据状态 | 规范模板固定为 `D`；复制实例单独声明 |

项目实例还必须包含项目/产品标识、工作产品标识、版本、状态、提供者、评审者、批准者、日期、适用性、基线和归档位置。

## 4. 占位符与完成规则

- 未填写字段使用 `[填写：字段说明]`。
- 暂不能确定但允许继续工作的项目值使用 `[待确认：责任角色；截止日期；关闭条件]`。
- `不适用` 必须给出理由和批准人。
- 进入批准基线前，不允许保留 `[填写：...]`；保留的待确认项必须有风险接受和关闭计划。
- 模板正文不得使用“最终版”“最新版”等不可复现状态。

## 6. 复制与裁剪

1. 从受控标签或提交复制模板，不从旧项目实例反向复制。
2. 复制后把模板标识保留为“模板来源”，另分配项目工作产品标识。
3. 根据项目类型、产品边界、合同、法规、安全和网络暴露面记录适用性。
4. 合并模板只能合并载体，不能合并工作产品责任；被合并工作产品仍分别标识和批准。
5. 裁剪不能删除来源、责任、追溯、评审、版本、变更或归档控制。
6. 客户交付、量产、安全或网络安全相关项目不得裁剪双向追溯、配置基线、缺陷闭环和发布批准。

## 7. 评审与基线

- 内容评审必须包含工作产品提供方和下游使用方。
- 质量负责人检查过程与证据完整性，不替代技术负责人批准专业内容。
- 配置管理员检查标识、版本、状态、基线、变更和归档。
- 模板发布执行 V4-G1–V4-G5；项目实例执行产品生命周期 G0–G6。

## 8. 自动检查合同

`scripts/check_template_assets.py` 至少检查：

- 18 个模板文件和标识均存在且唯一；
- P0 为 13 项、P1 为 5 项；
- 通用元数据和必要章节存在；
- 最终责任角色来自权威角色表；
- MK8 试点至少复制 9 项必须模板；
- 模板目录路径与实际文件一致。
