## 3. 适用范围

### Automotive SPICE 与 ISO/IEC 33020

- Automotive SPICE 4.0 用于建立过程目的、过程结果、基础实践、工作产品和评估指标的工程索引。
- ISO/IEC 33020:2019 用于建立能力等级、过程属性和评级证据口径。
- v0.3 重点覆盖 v0.2 已批准的 11 个总控/专业过程，目标是支持 PA1.1、PA2.1、PA2.2、PA3.1 和 PA3.2 差距分析。
- 文档映射不等于过程执行证据，也不形成任何能力等级评级。

### ISO 26262

- 主要映射 Parts 2、3、4、6、7、8 和 9，覆盖安全管理、概念、系统、软件、生产运行接口、支撑过程和安全分析。
- Parts 1 和 10 用于术语和应用指导。
- Part 5 硬件开发在 v0.3 中只映射到 `EXT-HW` 外部责任、接口和证据要求，不扩展为 MEES 硬件详细过程。
- Parts 11 和 12 仅在半导体或摩托车项目适用时登记为专项扩展。

### IEC 62443

- 以 IEC 62443-4-1 的安全产品开发生命周期为主轴。
- IEC 62443-3-2 承接系统风险评估、区域/通道和目标安全等级接口。
- IEC 62443-3-3 与 4-2 承接系统和组件技术安全要求，并映射到系统、软件、测试、配置、漏洞和发布过程。
- 运营方和服务提供方管理要求不在 v0.3 主范围内；需要时登记接口和缺口，不推定由产品开发过程直接覆盖。

### ISO 9001

- 以 ISO 9001:2015 和 Amd 1:2024 为当前发布基线。
- 映射顾客与相关方要求、组织环境、策划、支持、运行、绩效评价和改进与 MEES 的接口。
- 在研后续版本只进入观察表；正式发布并完成影响分析前，不修改已批准映射基线。

## 4. 标准适用性矩阵

| MEES 过程域 | ASPICE / 33020 | ISO 26262 | IEC 62443 | ISO 9001 |
|---|---|---|---|---|
| 产品规划 | 支撑业务目标、需求来源和发布输入 | 安全相关产品边界和使用场景接口 | 产品安全范围和预期使用环境接口 | 顾客、相关方和策划接口 |
| 项目管理 | 项目管理和能力属性证据 | 安全计划、独立性和确认措施接口 | 安全开发计划和责任接口 | 策划、风险机会和资源接口 |
| 需求治理 | SYS/SWE 需求治理和追溯 | 安全需求层级和追溯 | 安全需求及风险处置追溯 | 顾客与设计开发输入控制 |
| 系统工程 | 系统需求和架构过程 | 概念、系统开发和安全分析 | 系统风险、区域/通道和系统要求 | 设计开发控制接口 |
| 软件工程 | 软件需求、设计、实现和单元验证 | 软件安全需求和开发 | 安全设计、实现和组件要求 | 受控开发和变更接口 |
| 集成、验证与确认 | 软件/系统集成测试和测试过程 | 安全验证、确认和证据接口 | 安全验证、漏洞和负向测试 | 监视测量、放行和不合格输出接口 |
| 配置与变更 | SUP.8、SUP.9、SUP.10 及 PA2.2 | 支撑过程和安全异常控制 | 配置、缺陷、漏洞、补丁和变更 | 形成文件的信息、不合格和纠正措施 |
| 发布与反馈 | 发布和项目反馈接口 | 生产发布、运行、服务和退役接口 | 安全发布、补丁、通知和生命周期结束 | 产品放行、顾客反馈和持续改进 |

矩阵中的“接口”不表示直接或完全覆盖；详细关系必须在专项映射中使用统一关系类型作出结论。

## 5. 统一映射字段

| 字段 | 必填 | 说明 |
|---|---|---|
| 映射编号 | 是 | 使用 `MAP-ASP-*`、`MAP-CAP-*`、`MAP-FS-*`、`MAP-CS-*` 或 `MAP-QMS-*` |
| 来源编号 | 是 | 引用第 2 节的 `SRC-*` 登记项 |
| 标准引用标识 | 是 | 只记录过程号、章节号或要求标识，不复制标准原文 |
| 工程解释 | 是 | 使用 MEES 原创语言说明工程关注点 |
| MEES 过程 | 是 | 指向唯一过程文档和适用活动 |
| MEES 工作产品 | 是 | 指向工作产品唯一责任矩阵中的产物或治理证据 |
| 最终责任角色 | 是 | 使用核心过程总览的权威角色名称 |
| 关系类型 | 是 | 直接覆盖、支撑覆盖、部分覆盖、不适用或缺口 |
| 关系理由 | 是 | 说明范围、边界和相邻过程接口 |
| 最小预期证据 | 是 | 说明文档、记录、工具数据、访谈或实施结果 |
| 证据状态 | 是 | 使用第 6 节定义的 D/S/P/I 状态 |
| 门禁 | 否 | 关联 G0–G6 或 V3-G0–V3-G5 |
| 差距编号 | 条件必填 | 部分覆盖或缺口必须关联 `GAP-V03-*` |
| 处置与目标版本 | 条件必填 | 记录责任角色、关闭判据和计划版本 |
| 评审状态 | 是 | 草稿、已复核、已批准或已废止 |

## 6. 证据状态模型

| 状态 | 名称 | 判定边界 |
|---|---|---|
| D | 文档定义证据 | 过程、规则、模板或映射已经定义，但没有项目执行记录 |
| S | 模拟演练证据 | 桌面走查或模拟角色评审已经完成，明确不是真实项目部署 |
| P | 项目实施证据 | 真实项目按过程执行并保留受控记录、工具数据和结果 |
| I | 独立评价证据 | 具备独立性的真实评审、审核或评估已经形成可审计结论 |

证据状态可以组合，例如 `D+S`。不得以高状态替代低状态的内容完整性，也不得把 `D` 或 `S` 表述为 `P` 或 `I`。v0.3 文档建设的默认上限为 `D+S`。

## 7. 术语口径

| 术语 | MEES 统一含义 |
|---|---|
| 标准引用 | 指向官方标准中的过程号、章节号或要求标识，不含大段原文 |
| 工程映射 | 标准关注点与 MEES 过程、工作产品、角色、门禁和证据之间的解释关系 |
| 覆盖 | 在指定范围和证据状态下存在可说明的承接关系，不等于符合性结论 |
| 适用性 | 基于产品、项目、法规、合同和裁剪条件判断某项标准关注点是否进入范围 |
| 能力等级 | ISO/IEC 33020 框架下基于过程属性评级形成的结果，不由文档数量推定 |
| 工作产品 | 由唯一过程和最终责任角色负责的受控工程产物 |
| 证据 | 支持过程执行、工作产品状态或评价结论的可审计信息 |
| 差距 | 当前过程、工作产品、角色、接口或证据不足以支撑目标映射的状态 |
| 模拟审计 | 用于验证流程和记录结构的演练，不构成真实独立评价或认证 |

## 8. 版本变更观察

| 观察项 | 当前处理 | 触发动作 |
|---|---|---|
| ISO 9001 后续版本 | 仍使用 2015 + Amd 1:2024 | 正式发布后登记新来源、完成影响分析，再决定 v0.3 修订或后续版本迁移 |
| IEC 62443-4-1 第 2 版 | 不纳入当前基线 | 正式发布后比较安全生命周期差异 |
| ISO 26262 后续修订 | 继续使用 2018 系列 | 任一适用部分正式替代后开展逐部分影响分析 |
| Automotive SPICE 扩展模型 | 按专项需要作为补充来源 | 不得覆盖或改变 Automotive SPICE 4.0 主模型引用 |

## 9. 版权与使用边界

- 仓库只保存原创工程解释、引用标识、适用性结论和证据索引。
- 不提交购买版标准全文、受限截图、长段摘录或非授权译文。
- 官方网页链接用于来源和版本核验，不表示网页摘要足以完成符合性判断。
- 详细映射评审人必须能够合法访问所引用版本的标准正文。
- 对外审核或认证时，以合同、法规和审核机构确认的标准版本为准。
