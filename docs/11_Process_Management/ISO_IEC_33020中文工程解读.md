# ISO/IEC 33020 中文工程解读

> 本文是原创工程解读，不是标准原文、官方译文或逐条翻译。

## 1. 标准作用

ISO/IEC 33020 提供过程能力测量框架，用于评价某一过程从“是否完成”到“是否受控、标准化、可预测并持续优化”的能力水平。

## 2. 两个评价维度

### 过程维度

回答“评价哪个过程”，例如：需求分析、架构设计、项目管理、配置管理、测试等。

### 能力维度

回答“该过程实施得有多好”。能力不是产品功能数量，也不是团队人数，而是过程稳定、受控、可重复和可改进的程度。

## 3. 能力等级概览

| 等级 | 名称 | 工程含义 |
|---|---|---|
| CL0 | 不完整级 | 过程未实施，或无法证明达到目的 |
| CL1 | 已执行级 | 能够完成过程并产生预期结果 |
| CL2 | 已管理级 | 过程经过计划、监控，工作产品受控 |
| CL3 | 已建立级 | 组织标准过程得到定义、裁剪和部署 |
| CL4 | 可预测级 | 使用量化目标和统计方法控制过程 |
| CL5 | 创新级 | 基于业务目标持续改进与创新 |

## 4. 过程属性

| 属性 | 关注点 |
|---|---|
| PA1.1 过程执行 | 过程目的和结果是否实现 |
| PA2.1 执行管理 | 是否计划、监控、调整并明确责任资源 |
| PA2.2 工作产品管理 | 输出物是否定义、评审、配置和受控 |
| PA3.1 过程定义 | 是否形成组织标准过程及裁剪规则 |
| PA3.2 过程部署 | 项目是否按裁剪后的标准过程执行 |
| PA4.1 定量分析 | 是否建立量化目标和测量方法 |
| PA4.2 定量控制 | 是否识别波动并保持过程稳定 |
| PA5.1 过程创新 | 是否识别并评价改进机会 |
| PA5.2 过程优化 | 是否部署、验证并固化有效改进 |

## 5. N/P/L/F 评级

| 等级 | 中文 | 常见解释 |
|---|---|---|
| N | 未达到 | 几乎没有证据 |
| P | 部分达到 | 有实施，但覆盖有限或不稳定 |
| L | 大部分达到 | 总体达成，仍有少量明显弱点 |
| F | 完全达到 | 系统性、充分且无重大弱点 |

评估不能只看“有没有文档”，而应综合访谈、记录、工具数据、版本历史、评审证据和实际结果。

## 6. 从 CL1 到 CL3 的工程差异

- CL1：项目把事情做完。
- CL2：项目知道谁来做、何时做、如何监控，并控制输出物。
- CL3：组织定义统一方法，各项目按规则裁剪并形成可复用经验。

## 7. 与 ASPICE 的关系

ASPICE 定义汽车研发中需要评估的过程和实践；ISO/IEC 33020 提供能力等级和测量逻辑。两者结合后，既能评价“做什么”，也能评价“做到什么程度”。

## 8. 在 MEES 中的落地

每个过程文档至少包含：

- 目的和预期结果
- 输入、活动和输出
- 角色与责任
- 工作产品及配置规则
- 评审与质量保证
- 度量指标
- 裁剪规则
- 证据清单
- 对应能力属性
