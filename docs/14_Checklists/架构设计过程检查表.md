# 架构设计过程检查表

## 1. 基础完整性

- [ ] 架构治理范围覆盖系统架构、软件架构、接口和关键机制的跨层一致性。
- [ ] 架构过程承接需求基线，并向实现、集成和验证提供输入。
- [ ] 架构决策、风险和约束有记录机制。

## 2. 输入输出

- [ ] 输入包含系统需求、软件需求、安全要求、网络安全要求、平台约束和历史问题。
- [ ] 输出只包含治理规则、跨层决策、评审记录、设计基线批准包、追溯矩阵和风险问题台账。
- [ ] 系统架构、接口控制和软件架构分别引用系统工程和软件工程的受控输出。

## 3. 角色职责

- [ ] 架构治理负责人只对治理规则、跨层决策、追溯、评审和基线批准包最终负责。
- [ ] 系统负责人对系统架构和接口控制最终负责，软件负责人对软件架构和详细设计最终负责。
- [ ] 安全、网络安全、开发、测试和质量角色参与相关架构评审。

## 4. 工作产品

- [ ] 跨层架构决策台账包含问题、备选方案、专业意见、决策、影响、风险和状态。
- [ ] 跨层追溯矩阵连接需求、系统元素、`EXT-HW`、软件元素、测试项和风险。
- [ ] 专业架构工作产品的内容检查由系统工程和软件工程检查承担。

## 5. 评审与批准

- [ ] 架构评审覆盖需求分配、接口一致性、关键风险和可测试性。
- [ ] 关键架构决策由架构、开发、测试和质量负责人确认。
- [ ] 安全和网络安全机制由对应领域负责人批准。

## 6. 配置与变更

- [ ] 架构治理规则、跨层决策、基线批准包、评审记录和追溯矩阵纳入配置管理。
- [ ] 架构变更分析对需求、实现、测试、发布和风险的影响。
- [ ] 架构基线与需求基线和验证策略保持一致。

## 7. 度量指标

- [ ] 需求架构覆盖率可追踪。
- [ ] 架构评审问题关闭率可统计。
- [ ] 接口变更次数可统计。
- [ ] 关键技术风险关闭率可见。

## 8. 实施证据

- [ ] 架构治理产物有版本记录，专业架构工作产品有受控引用。
- [ ] 架构评审问题有关闭证据。
- [ ] 架构追溯矩阵覆盖关键需求和测试项。
- [ ] 技术风险分析和缓解措施有记录。

## 9. ASPICE / ISO 33020 关注点

- [ ] 支撑 SYS.3 系统架构设计。
- [ ] 支撑 SWE.2 软件架构设计。
- [ ] 覆盖 PA1.1、PA2.2 和 PA3.1。
- [ ] 为集成测试和系统测试提供可验证设计输入。

## 10. 裁剪规则

- [ ] 小型软件变更裁剪后仍记录受影响组件、接口、风险和验证影响。
- [ ] 平台级、安全相关或跨团队接口变更未裁剪架构评审和接口基线。
- [ ] 裁剪结论有架构负责人和质量负责人确认。
