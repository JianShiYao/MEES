# 验证确认过程检查表

## 1. 基础完整性

- [ ] 验证范围统筹单元、集成、系统、确认、回归和发布前验证，执行职责已分配到专业过程。
- [ ] 验证过程承接需求和架构基线。
- [ ] 验证结果可支撑发布决策。

## 2. 输入输出

- [ ] 输入包含需求/架构基线、单元结果、分层测试证据包、风险和发布计划。
- [ ] 输出包含总体验证策略、跨层覆盖汇总、证据索引、G5 记录和发布验证建议。
- [ ] 分层测试工作产品由集成与测试过程提供，验证确认过程不重复拥有。

## 3. 角色职责

- [ ] 测试负责人对总体验证策略、跨层覆盖、G5 结论和发布验证建议最终负责。
- [ ] 测试工程师提供分层证据并支持覆盖、缺陷和风险汇总。
- [ ] 开发、系统、配置和质量角色参与缺陷分析、版本确认和发布判断。

## 4. 工作产品

- [ ] 总体验证策略定义层级、目标、任务分配、独立性、资源和总准入准出规则。
- [ ] 分层验证证据索引引用受控的单元、集成、软件、系统和确认测试报告。
- [ ] 跨层汇总包含覆盖缺口、缺陷、偏差、豁免和遗留风险。
- [ ] G5 记录和发布验证建议包含门禁结论、条件、责任人和批准角色。

## 5. 评审与批准

- [ ] 总体验证策略经过测试、项目、工程和质量代表评审。
- [ ] 专业过程提供的分层测试证据覆盖需求、接口、异常、边界、回归和风险场景。
- [ ] G5 记录和发布验证建议经过测试、工程和质量负责人确认。

## 6. 配置与变更

- [ ] 总体验证策略、覆盖汇总、证据索引、G5 记录和发布建议纳入配置管理。
- [ ] 分层用例、脚本、环境、数据、执行记录和报告由产生它们的专业过程受控。
- [ ] 需求或架构变更后执行跨层验证影响分析。

## 7. 度量指标

- [ ] 跨层需求验证覆盖率可追踪。
- [ ] 分层证据接收完成率和 G5 行动项关闭率可统计。
- [ ] 遗留风险批准状态和发布阻塞缺陷数可见。

## 8. 实施证据

- [ ] 总体验证策略和任务分配可追溯到需求、设计及项目目标。
- [ ] 单元和分层测试报告通过证据索引可复核。
- [ ] 覆盖、缺陷、偏差和遗留风险汇总可追溯到原始证据。
- [ ] G5 记录和发布验证建议明确是否满足发布准则。

## 9. ASPICE / ISO 33020 关注点

- [ ] 支撑 SWE.4 软件单元验证。
- [ ] 支撑 SWE.5 软件集成测试。
- [ ] 支撑 SWE.6 软件测试。
- [ ] 支撑 SYS.4 系统集成测试和 SYS.5 系统测试。
- [ ] 覆盖 PA1.1、PA2.1 和 PA2.2。

## 10. 裁剪规则

- [ ] 探索性原型裁剪后仍保留关键验收结果和已知风险。
- [ ] 客户交付、安全相关、网络安全相关或量产发布未裁剪需求覆盖和缺陷闭环。
- [ ] 裁剪结论有测试负责人和质量负责人确认。
