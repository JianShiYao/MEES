# 验证确认过程检查表

## 1. 基础完整性

- [ ] 验证范围覆盖单元、集成、系统、确认、回归和发布前验证。
- [ ] 验证过程承接需求和架构基线。
- [ ] 验证结果可支撑发布决策。

## 2. 输入输出

- [ ] 输入包含需求基线、验收准则、架构说明、接口说明、构建包和历史缺陷。
- [ ] 输出包含测试计划、测试用例、执行记录、缺陷记录、测试报告和发布验证建议。
- [ ] 测试输出与需求、缺陷、构建版本和发布候选保持追溯。

## 3. 角色职责

- [ ] 测试负责人负责验证策略、资源计划和测试报告。
- [ ] 测试工程师负责测试设计、执行和维护。
- [ ] 开发、系统、配置和质量角色参与缺陷分析、版本确认和发布判断。

## 4. 工作产品

- [ ] 测试计划定义范围、层级、环境、准入准出、资源和进度。
- [ ] 测试用例包含前置条件、步骤、输入、预期结果和追溯关系。
- [ ] 测试执行记录包含版本、环境、结果、日志、偏差和执行人。
- [ ] 缺陷记录包含严重度、复现步骤、影响、责任人和关闭证据。

## 5. 评审与批准

- [ ] 测试计划经过测试、项目、工程和质量代表评审。
- [ ] 测试用例覆盖需求、接口、异常、边界、回归和风险场景。
- [ ] 发布前测试报告经过测试负责人和质量负责人确认。

## 6. 配置与变更

- [ ] 测试计划、用例、脚本、环境配置、测试数据和报告纳入配置管理。
- [ ] 需求或架构变更后执行测试影响分析。
- [ ] 测试版本、环境和构建包可复现。

## 7. 度量指标

- [ ] 需求测试覆盖率可追踪。
- [ ] 测试通过率可统计。
- [ ] 缺陷发现和关闭趋势可见。
- [ ] 自动化测试占比和回归完成率可统计。

## 8. 实施证据

- [ ] 测试计划和准入准出准则有版本记录。
- [ ] 测试用例、脚本和执行记录可追溯。
- [ ] 缺陷关闭有验证证据。
- [ ] 测试报告包含覆盖率、缺陷状态、风险和结论。

## 9. ASPICE / ISO 33020 关注点

- [ ] 支撑 SWE.4 软件单元验证。
- [ ] 支撑 SWE.5 软件集成测试。
- [ ] 支撑 SWE.6 软件测试。
- [ ] 支撑 SYS.4 系统集成测试和 SYS.5 系统测试。
- [ ] 覆盖 PA1.1、PA2.1 和 PA2.2。

## 10. 裁剪规则

- [ ] 探索性原型裁剪后仍保留关键验收结果和已知风险。
- [ ] 客户交付、安全相关、网络安全相关或量产发布未裁剪需求覆盖和缺陷闭环。
- [ ] 裁剪结论有测试负责人和质量负责人确认。
