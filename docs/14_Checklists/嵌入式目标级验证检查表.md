# 嵌入式目标级验证检查表

本检查表配合[嵌入式目标级验证方法](../05_Test_Engineering/嵌入式目标级验证方法.md)使用，用于确认分层验证证据充分且不被误用。安全相关（SIL）项目所有"不适用"必须记录理由与批准人。

## 1. 证据层级与不可互换性

- [ ] 每条验证证据标注层级（主机/SIL/PIL/目标硬件/HIL）与证据状态（D/S/P）。
- [ ] 未执行的目标硬件/HIL 验证保持"未执行"，未被主机/SIL 通过推定替代。
- [ ] 安全相关功能具备目标硬件或 HIL 层级的验证计划。

## 2. 工具链与制品身份

- [ ] 记录编译器/链接器版本、构建变体（App/Boot）与制品哈希。
- [ ] 构建可在受控环境复现。
- [ ] 启动代码与内存布局纳入受控配置。

## 3. 代码质量

- [ ] 编码规则与静态分析已执行，偏差有批准。
- [ ] 单元验证覆盖正常/边界/异常，结构覆盖目标已定义。

## 4. 动态与实时

- [ ] 接口、时序/WCET 假设、资源/栈、并发、中断已验证或登记假设。
- [ ] 看门狗、启动/升级与故障恢复行为已验证。

## 5. 安全相关验证

- [ ] 故障注入、边界值与降级/安全状态进入及保持已验证或计划。
- [ ] 诊断覆盖已评估。

## 6. 可复现性与追溯

- [ ] 测试环境、桩件边界、测量设备与工具版本已记录。
- [ ] 需求→架构→单元→测试（含层级）→缺陷→构建→发布双向可追溯。
- [ ] 追溯生成器无未处置的 `TRC-NO-VERIF`；`TRC-TEST-NOTRUN` 项有执行计划。

## 7. 结论

- [ ] 缺目标/HIL 执行证据时，验证状态与发布建议保持"未执行/No-Go"，无虚假绿灯。
