# 项目管理过程检查表

## 1. 基础完整性

- [ ] 项目目标、范围、交付物和成功准则清晰。
- [ ] 项目阶段、里程碑、资源和关键约束已定义。
- [ ] 项目过程与需求、架构、验证、发布和配置管理接口明确。

## 2. 输入输出

- [ ] 输入包含产品路线图、客户需求、项目章程、资源约束和历史经验。
- [ ] 输出包含项目计划、风险问题台账、状态报告、里程碑评审和项目复盘。
- [ ] 每个输出物有责任人、维护位置和更新频率。

## 3. 角色职责

- [ ] 项目经理、产品负责人、工程负责人、测试负责人、质量负责人和配置管理员职责明确。
- [ ] 风险、问题、依赖和决策均有责任人。
- [ ] 关键评审和变更批准角色无缺失。

## 4. 工作产品

- [ ] 项目章程已批准。
- [ ] 项目计划覆盖进度、资源、质量活动和交付物。
- [ ] 风险与问题台账包含等级、缓解措施、状态和关闭证据。
- [ ] 状态报告能反映进度、质量、风险、变更和阻塞。

## 5. 评审与批准

- [ ] 项目计划经过项目、产品、工程和质量代表评审。
- [ ] 关键里程碑评审检查需求、设计、验证、缺陷、风险和配置状态。
- [ ] 范围、里程碑、成本或质量目标变化有正式批准记录。

## 6. 配置与变更

- [ ] 项目计划、里程碑基线、风险台账和评审记录纳入配置管理。
- [ ] 变更记录包含原因、影响分析、批准人和实施状态。
- [ ] 项目状态与配置基线、发布计划保持一致。

## 7. 度量指标

- [ ] 里程碑达成率有数据来源。
- [ ] 需求完成率、缺陷关闭率和风险关闭率可追踪。
- [ ] 工程证据完整率可用于质量门禁。

## 8. 实施证据

- [ ] 项目章程和项目计划可追溯到版本记录。
- [ ] 风险、问题、依赖和决策台账持续维护。
- [ ] 里程碑评审行动项有关闭证据。
- [ ] 项目复盘记录包含改进项。

## 9. ASPICE / ISO 33020 关注点

- [ ] 支撑 MAN.3 项目管理。
- [ ] 覆盖 PA1.1 过程执行。
- [ ] 覆盖 PA2.1 执行管理。
- [ ] 覆盖 PA2.2 工作产品管理。

## 10. 裁剪规则

- [ ] 小型项目裁剪后仍保留目标、范围、责任人、里程碑和风险记录。
- [ ] 客户交付、安全相关或高风险项目未裁剪质量门禁和配置管理活动。
- [ ] 裁剪结论有项目经理和质量负责人确认。
