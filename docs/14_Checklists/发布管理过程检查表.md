# 发布管理过程检查表

## 1. 基础完整性

- [ ] 发布范围、版本号、目标环境、交付对象和发布时间明确。
- [ ] 发布过程承接项目计划、配置基线、测试报告和质量门禁。
- [ ] 发布结论可支撑客户交付、量产导入或内部使用。

## 2. 输入输出

- [ ] 输入包含发布计划、需求状态、缺陷状态、构建产物、测试报告和遗留风险。
- [ ] 输出包含发布候选包、发布说明、发布评审记录、发布归档和发布后反馈。
- [ ] 发布输出与配置基线、版本标签和测试报告保持一致。

## 3. 角色职责

- [ ] 发布负责人组织发布计划、评审、归档和沟通。
- [ ] 配置管理员维护版本、基线、标签、构建和交付包。
- [ ] 项目、工程、测试和质量负责人参与发布批准。

## 4. 工作产品

- [ ] 发布计划包含范围、版本号、时间、对象、准入准出和责任人。
- [ ] 发布候选包包含构建产物、配置、依赖、校验信息和说明。
- [ ] 发布说明包含新增、修复、已知问题、兼容性和升级/回退说明。
- [ ] 发布归档包含标签、构建日志、测试报告、交付包和证据。

## 5. 评审与批准

- [ ] 发布评审确认需求范围、变更范围、缺陷状态、验证结果和遗留风险。
- [ ] 回退方案已定义并可执行。
- [ ] 阻塞级缺陷未关闭时有正式豁免和批准记录。
- [ ] 客户交付和量产发布有项目、工程、测试、配置和质量负责人批准。

## 6. 配置与变更

- [ ] 发布包、版本标签、构建日志、配置文件、发布说明和批准记录归档。
- [ ] 发布后变更产生新的版本、补丁或变更记录。
- [ ] 发布候选与最终交付包可校验一致性。

## 7. 度量指标

- [ ] 发布成功率可统计。
- [ ] 发布回退次数可统计。
- [ ] 发布阻塞缺陷数可见。
- [ ] 发布证据完整率可检查。
- [ ] 发布后缺陷逃逸率可跟踪。

## 8. 实施证据

- [ ] 发布计划和范围冻结记录存在。
- [ ] 构建日志、标签、校验信息和交付包可追溯。
- [ ] 测试报告、缺陷状态和遗留风险记录完整。
- [ ] 发布评审和批准记录完整。

## 9. ASPICE / ISO 33020 关注点

- [ ] 支撑 SPL.2 产品发布。
- [ ] 与 SUP.8 配置管理、SUP.9 问题解决管理、SUP.10 变更请求管理接口清楚。
- [ ] 覆盖 PA1.1、PA2.1 和 PA2.2。

## 10. 裁剪规则

- [ ] 内部探索版本裁剪后仍保留版本号、构建来源、交付内容和已知风险。
- [ ] 客户交付、量产和安全相关版本未裁剪发布评审、版本标签、测试报告和归档证据。
- [ ] 裁剪结论有发布负责人和质量负责人确认。
