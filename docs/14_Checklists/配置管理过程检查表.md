# 配置管理过程检查表

## 1. 基础完整性

- [ ] 配置项范围覆盖需求、架构、设计、源码、测试、构建、发布和质量证据。
- [ ] 配置管理过程支撑全部主流程。
- [ ] 配置项、基线、变更、状态记录、审计和归档规则明确。

## 2. 输入输出

- [ ] 输入包含项目计划、发布计划、工程工作产品、变更申请、构建配置和组织规则。
- [ ] 输出包含配置管理计划、配置项清单、基线记录、变更记录、状态报告、审计记录和发布归档。
- [ ] 配置输出可支撑发布复现、质量审核和过程评估。

## 3. 角色职责

- [ ] 配置管理员负责配置计划、清单、基线、标签和归档。
- [ ] 项目经理确认基线计划、变更优先级和发布节奏。
- [ ] 工程、测试、发布和质量负责人参与影响分析、审计和发布确认。

## 4. 工作产品

- [ ] 配置管理计划定义配置项范围、工具、分支策略、基线规则和职责。
- [ ] 配置项清单包含编号、名称、类型、责任人、位置、版本和状态。
- [ ] 基线记录包含基线名称、内容、版本、建立时间、批准人和用途。
- [ ] 配置状态报告反映当前基线、开放变更、发布候选和风险。

## 5. 评审与批准

- [ ] 配置管理计划经过项目、工程、发布和质量负责人评审。
- [ ] 正式基线明确范围、版本、批准人和使用场景。
- [ ] 影响需求、架构、验证或发布的配置变更完成影响分析和批准。
- [ ] 发布基线经过配置审计。

## 6. 配置与变更

- [ ] 正式配置项纳入受控存储位置。
- [ ] 配置项变更保留提交记录、评审记录、原因、影响分析、批准和验证状态。
- [ ] 发布后配置变更产生新版本、补丁或变更记录。

## 7. 度量指标

- [ ] 配置项受控率可统计。
- [ ] 基线按期建立率可统计。
- [ ] 变更关闭周期可统计。
- [ ] 未批准变更数可见。
- [ ] 构建可复现率和发布归档完整率可检查。

## 8. 实施证据

- [ ] 配置管理计划和配置项清单有版本记录。
- [ ] 分支、标签、提交、评审和合并记录可追溯。
- [ ] 需求、架构、验证和发布基线记录完整。
- [ ] 配置审计问题有整改和关闭证据。

## 9. ASPICE / ISO 33020 关注点

- [ ] 支撑 SUP.8 配置管理。
- [ ] 与 SUP.10 变更请求管理、SUP.9 问题解决管理和 SPL.2 产品发布接口清楚。
- [ ] 覆盖 PA1.1、PA2.1、PA2.2 和 PA3.2。

## 10. 裁剪规则

- [ ] 小型探索项目裁剪后仍保留配置项清单、版本记录和发布归档。
- [ ] 客户交付、量产、安全相关或网络安全相关项目未裁剪基线管理、变更记录、配置审计和发布归档。
- [ ] 外部工具或客户平台中的证据与 MEES 配置证据有映射关系。
