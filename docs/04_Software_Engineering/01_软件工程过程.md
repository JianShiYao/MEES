# 软件工程过程

> 文档编号：MEES-PRO-401
> 版本：v0.2.0
> 状态：已批准（模拟审计）
> 所有者：软件负责人
> 最后更新：2026-07-14

## 1. 目的

定义软件需求分析、软件架构、详细设计、实现、代码评审和单元验证的连续工程过程，确保软件增量满足系统分配要求并具备受控集成条件。

## 2. 适用范围

适用于嵌入式软件、固件、驱动、中间件、应用、启动程序、诊断和通信模块。第三方软件和自动生成代码也必须定义准入、验证和配置控制方式。

## 3. 流程位置

软件工程承接系统分配需求、系统接口和项目迭代目标，向集成与测试过程交付受控软件增量、单元验证结果、构建信息、集成顺序和已知限制。

## 4. 输入

| 输入 | 来源 |
|---|---|
| 软件分配需求、系统架构和接口 | 系统工程 |
| 软件范围、迭代目标和资源约束 | 项目管理 |
| 安全、网络安全、编码和平台约束 | 领域工程 / 工程平台 |
| 复用组件、第三方软件和工具链 | 平台 / 供应商 / DevOps |
| 问题、变更和历史缺陷 | 变更与问题管理 |

## 5. 活动

1. 分析系统分配需求，形成软件需求、属性、验证准则和双向追溯。
2. 设计软件架构，定义组件、职责、接口、并发、资源、初始化和错误处理机制。
3. 完成详细设计，定义单元行为、数据、算法、状态、边界条件和异常路径。
4. 建立需求到架构、详细设计、实现单元和验证项的追溯。
5. 按编码规则实现，执行静态检查、代码评审和持续构建。
6. 设计并执行单元验证，覆盖正常、边界、异常和适用的结构覆盖目标。
7. 汇总软件增量、单元结果、开放问题和集成说明，通过 G4 准入评审。

## 6. 输出与工作产品

| 工作产品 | 最小要求 |
|---|---|
| 软件需求规格 | 标识、来源、行为、接口、约束、验证准则和追溯 |
| 软件架构说明 | 组件、职责、依赖、接口、运行机制、资源和风险 |
| 软件详细设计 | 单元、算法、状态、数据、边界、异常和设计理由 |
| 源码与构建配置 | 版本、分支/提交、依赖、工具链和构建方式 |
| 代码评审与静态检查记录 | 规则、发现、结论、豁免和关闭证据 |
| 单元验证规格与报告 | 用例、环境、版本、结果、覆盖和缺陷 |
| 软件交付说明 | 软件基线、接口、集成顺序、限制和开放问题 |

## 7. 角色与职责

| 角色 | 职责 |
|---|---|
| 软件负责人 | 对软件需求、设计、实现策略和增量完整性负责 |
| 软件架构师 | 对组件分解、接口、运行机制和关键决策负责 |
| 软件开发工程师 | 对详细设计、实现、代码质量和单元验证负责 |
| 系统工程师 | 确认系统分配、接口和系统约束满足情况 |
| 测试工程师 | 评估可测试性、集成条件和软件测试覆盖 |
| 配置管理员 | 维护源码、工具链、构建、依赖和软件基线 |
| 质量负责人 | 检查评审、追溯、准入条件和豁免证据 |

## 8. 流程图

```mermaid
flowchart LR
    A[系统分配需求] --> B[软件需求分析]
    B --> C[软件架构]
    C --> D[详细设计]
    D --> E[实现与代码评审]
    E --> F[单元验证]
    F --> G[持续构建与质量检查]
    G --> H{G4 集成准入}
    H -- 未通过 --> D
    H -- 通过 --> I[软件增量基线]
```

## 9. 评审与批准

- 软件需求评审覆盖系统分配一致性、可实现性、可验证性和追溯。
- 架构与详细设计评审覆盖接口、资源、并发、异常、安全、网络安全、可维护性和可测试性。
- G4 必须确认构建成功、代码评审完成、单元准出满足、阻塞问题清零、交付说明和集成环境就绪。

## 10. 配置与变更控制

软件需求、设计、源码、构建脚本、依赖、工具版本、评审和单元验证记录均为配置项。基线后变更必须关联问题或变更标识，并记录影响、提交、构建、回归和首次纳入版本。

## 11. 度量指标

| 指标 | 数据来源 |
|---|---|
| 软件需求设计覆盖率 | 软件追溯矩阵 |
| 代码评审完成率 | 代码评审平台 |
| 静态检查问题密度 | 静态分析报告 |
| 单元测试通过率 / 覆盖率 | 单元验证报告 |
| 构建成功率 | CI 平台 |
| 软件缺陷返工率 | 问题与变更台账 |

## 12. 裁剪规则

- 小型修改可合并软件架构和详细设计更新，但必须记录受影响需求、组件、接口、风险和验证范围。
- 自动生成代码可调整人工详细设计和代码评审方式，但不得裁剪生成模型/配置、工具版本、生成结果检查和验证。
- 安全、网络安全、平台核心或客户交付软件不得裁剪双向追溯、代码评审、单元验证和受控构建。

## 13. 实施证据

- 软件需求、架构、详细设计及评审记录。
- 需求—设计—实现—测试追溯记录。
- 源码提交、代码评审、静态检查和构建记录。
- 单元验证报告、覆盖结果、问题关闭和 G4 准入记录。
