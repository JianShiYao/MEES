# MK8 RSIIC V1 项目试点

> 状态：G0 准备中
> 试点类型：真实存量项目导入
> 证据边界：本地代码事实为 `P` 候选；未确认内容不作为项目证据
> 最后更新：2026-07-14

## 1. 目的与边界

以一个真实 BMS 固件项目验证 MEES 能否支持存量项目导入，并选择一个受控增量跑通 G0–G6。试点用于形成过程部署、裁剪、追溯和反馈证据，不评价产品功能安全等级，不替代网络安全评估、量产批准或客户验收。

原始代码位于本地 `examples/rsiic_-mk8_-v1/`，保留其独立 Git 历史，不纳入 MEES 公共仓库。MEES 只管理过程实例、证据索引和脱敏结论。

## 3. 导入判断

这是一个已有实现和提交历史、但过程证据未随代码交付的存量项目。不能把代码反向解释为已批准需求，也不能因为存在版本宏就认定 G6 已通过。试点采用“两条线并行”的方式：

1. **基线恢复线**：盘点代码、构建、模型、接口和版本信息，建立配置项与逆向追溯种子。
2. **受控增量线**：选择一个边界清晰的变更，按 MEES 从产品目标到发布完整执行 G0–G6。

推荐首个增量为“BMS 状态机关键告警接入与失效安全行为”。代码中的状态机仍声明为骨架，关键告警、SOC、功率和电流条件使用待集成桩函数，范围容易定位，也能覆盖系统需求、软件设计、实现、故障注入、回归和发布链路。该增量只是建议，必须由产品和工程角色确认后才能成为项目范围。

## 4. 初始产品机会草案

| G0 要素 | 当前草案 | 状态 |
|---|---|---|
| 目标用户 | 储能系统集成、调试和运维相关角色 | 基于接口与模块推断，待确认 |
| 业务问题 | BMS 状态行为、关键告警和外部协同需要形成可追溯、可验证的受控增量 | 待确认 |
| 候选范围 | 关键告警输入、状态转换、接触器安全输出、诊断与通信状态、相应测试 | 待确认 |
| 范围外 | 硬件详细开发、整包量产认可、ASIL/SL 认证声明、未关联的历史缺陷补录 | 建议，待确认 |
| 成功准则 | 需求到发布双向追溯完整；关键状态与故障路径测试通过；阻断缺陷为 0；构建与回退可复现 | 草案 |
| 候选版本 | 在当前 `1.1.15+13` 基线上建立独立试点版本 | 版本号待确认 |
| 功能安全 | BMS、高压接触器和保护功能具有潜在安全相关性 | 必须完成适用性决定，不作 ASIL 假设 |
| 网络安全 | 代码包含以太网、Modbus 和 OTA 路径 | 必须确认部署暴露面、鉴权和更新信任边界 |

## 5. 初始风险

| 风险 | 影响 | G0/G1 处置要求 |
|---|---|---|
| 产品需求、客户输入和验收准则未随代码提供 | 无法证明实现正确或范围完整 | 产品负责人确认来源并建立最小产品需求 |
| 代码仓库主动排除了 `docs/`、`test/`、工具和评审记录 | G2–G6 证据链中断 | 查找原始证据；找不到时按逆向建证计划补齐 |
| 状态机关键条件仍为固定返回值的桩 | 状态行为与真实系统脱节 | 作为候选增量纳入需求、设计和故障测试 |
| 存在 OTA、网络和现场总线接口 | 可能产生远程更新与访问风险 | G0 决定 IEC 62443 适用范围，G1 纳入安全活动 |
| 自动生成模型与手写代码混合 | 生成配置、评审和验证责任可能不清 | G1 裁剪中定义模型、生成代码和手写代码准出 |
| 原始代码含个人/组织信息及第三方组件 | 公开仓库泄露或许可不清 | 原始代码保持本地；公开前执行脱敏与许可证审查 |

## 7. 需要项目负责人确认的输入

1. 项目性质：客户交付、量产产品、内部平台还是学习样例。
2. 产品边界：MK8-P1 所在系统、主要用户、外部设备和运行场景。
3. 首个试点增量：是否采用“BMS 状态机关键告警接入与失效安全行为”。
4. 目标版本和时间窗口：当前基线之后的候选版本及期望完成时间。
5. 安全与网络安全：现有 ASIL/安全等级、客户约束、OTA 和网络暴露方式；没有正式结论时明确写“待分析”。
6. 真实角色：产品负责人、项目经理、系统负责人、软件负责人、测试负责人、质量负责人和最终 G0 决策人。

上述输入确认后，先形成 G0 产品机会评审记录；G0 通过后再建立 G1 项目章程、项目计划、裁剪记录和风险台账。

## 8. 过程运行顺序

| 阶段 | 本试点的主要动作 | 最小输出 |
|---|---|---|
| G0 | 确认产品问题、增量范围、价值、风险和适用标准 | 产品机会与真实决策记录 |
| G1 | 定义角色、里程碑、裁剪、配置和质量活动 | 项目章程、计划、裁剪、风险初版 |
| G2 | 建立产品/系统/软件需求及验收准则 | 需求基线、追溯和评审关闭记录 |
| G3 | 恢复系统上下文，定义接口、状态与软件设计 | 设计基线和评审记录 |
| G4 | 受控实现、评审、构建和单元验证 | 实现基线、构建与单元准入记录 |
| G5 | 执行软件/系统测试、故障注入和回归 | 测试报告、缺陷状态和发布建议 |
| G6 | 审计配置、生成候选包、回退和批准 | 标签、发布记录和归档清单 |
