# v0.6 BMS Agent 协作走查

> 走查编号：MEES-WT-004
> 版本：v0.6.0-dev
> 状态：已随 V6-G6 批准（模拟数据 S）
> 所有者：工程过程负责人
> 最后更新：2026-07-15

## 1. 目的与边界

使用 MK8 RSIIC V1 BMS 模拟项目验证六类 Agent 的公共协议、可回溯输出和权限边界。走查不包含真实目标测试、产品批准、独立 ASPICE 评价或真实 Obsidian 服务连接。

## 2. 可复现入口

```bash
python scripts/collect_metrics.py
python scripts/run_agent_pilot.py
```

输出位于 `build/agent-pilot/`，包括六份结构化结果、`audit.jsonl` 和 `summary.json`。生成目录不是受控源，结论必须回溯到受控输入和候选提交。

## 3. 六类任务

| Agent | 受控输入 | 检查重点 | 人工责任 |
|---|---|---|---|
| Requirement | `03_Requirements.md` | 需求标识与缺口建议 | 需求工程负责人 |
| Architecture | `04_Architecture.md` | 架构、详细设计与接口对象 | 系统负责人或软件负责人 |
| Review | `06_Release_Decision.md` | 未关闭检查项与评审建议 | 质量负责人 |
| Test | `05_Verification.md` | 测试标识、状态与发布边界 | 测试负责人 |
| ASPICE | 详细过程映射表 | 过程引用与映射覆盖 | 质量负责人 |
| Metrics | `build/metrics/metrics.json` | N/A 指标与解释责任 | 质量负责人 |

## 4. 走查结果

- 六类协议均以 `1.0.0` 版本运行，输出状态固定为 `human_review_required`。
- 输入证据为 `S`，输出保持 `S`；运行时拒绝 `P/I` 输入和路径穿越。
- Test Agent 识别“未执行”测试并保留主要发现。
- 所有协议禁止自动批准门禁、升级证据或发布产品。
- 汇总结论固定为 `No-Go`，原因是目标测试仍为计划未执行。

## 5. 错误边界

自动测试覆盖未知 Agent、证据升级、路径越界、O2 无连接降级、O3 确认不匹配、白名单外写入和回滚。Agent 发现只能作为草稿或建议，工作产品所有者负责接受、修改或拒绝。

## 6. 结论

WP6 的机器走查可复现，V6-G5 已具备人工评审输入；在人工确认前不得声明门禁 Go。MK8 产品发布保持 `No-Go`。
