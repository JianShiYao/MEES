# 系统工程过程

> 文档编号：MEES-PRO-301
> 版本：v0.2.0
> 状态：评审中
> 所有者：系统负责人
> 最后更新：2026-07-14

## 1. 目的

定义产品需求向系统需求、系统架构、接口和验证策略转化的方法，确保系统层技术要求完整、一致、可实现、可验证并可分配到软件、硬件和其他系统元素。

## 2. 适用范围

适用于嵌入式产品的系统需求分析、系统架构设计、内外部接口、功能分配、集成策略和系统验证准则。软件内部详细活动由软件工程过程负责；硬件详细开发过程不在 v0.2 范围内，硬件及外部组件分配以 `EXT-HW` 对象和受控外部规格/接口承接。

## 3. 流程位置

系统工程承接产品需求基线和项目约束，向软件工程提供软件分配需求、接口和资源约束，向集成与测试提供系统集成策略、系统测试项和确认准则。

## 4. 输入

| 输入 | 来源 |
|---|---|
| 产品目标、产品需求和使用场景 | 产品规划 / 需求管理 |
| 项目范围、里程碑和裁剪结果 | 项目管理 |
| 法规、安全、网络安全和质量约束 | 领域工程 |
| 平台、硬件、供应商和环境约束 | 平台 / 硬件 / 供应商 |
| 历史问题、现场数据和复用资产 | 质量 / 维护 / 工程平台 |

## 5. 活动

1. 分析产品需求和场景，建立系统边界、上下文、运行环境和外部接口。
2. 编写系统需求，定义功能、性能、接口、诊断、异常、资源和验证准则。
3. 分析系统需求的一致性、可行性、可验证性和双向追溯。
4. 设计系统架构，分解系统元素并将需求分配到软件需求或 `EXT-HW` 硬件/外部组件对象。
5. 定义接口、数据流、控制流、状态行为、降级策略和关键质量机制。
6. 建立系统集成策略、系统测试项、关键技术风险和架构决策。
7. 组织 G2/G3 评审，关闭问题后建立系统需求和系统设计基线。

## 6. 输出与工作产品

| 工作产品 | 最小要求 |
|---|---|
| 系统上下文与边界 | 外部参与者、环境、接口、假设和责任边界 |
| 系统需求规格 | 标识、来源、描述、属性、验证准则和追溯 |
| 系统架构说明 | 系统元素、职责、需求分配、行为和关键机制 |
| 接口控制说明 | 信号、协议、时序、单位、范围、错误和兼容性 |
| 硬件 / 外部分配清单 | `EXT-HW` 标识、责任方、受控规格/接口、版本和集成验证关系 |
| 系统集成与验证策略 | 集成顺序、环境、依赖、测试层级和准入条件 |
| 系统追溯矩阵 | 产品需求、系统需求、架构元素、测试项和问题关系 |

## 7. 角色与职责

| 角色 | 职责 |
|---|---|
| 系统负责人 | 对系统需求、系统架构和技术完整性负责 |
| 产品负责人 | 确认产品意图、优先级和验收目标 |
| 软件负责人 | 评估软件分配、接口、实现可行性和资源约束 |
| 硬件 / 外部组件责任方 | 接收 `EXT-HW` 分配并提供受控规格、接口和验证输入 |
| 测试负责人 | 确认可验证性、集成策略和系统测试范围 |
| 安全 / 网络安全负责人 | 确认领域需求、分析和技术措施分配 |
| 质量负责人 | 检查评审、基线、追溯和门禁证据 |

## 9. 评审与批准

- 系统需求评审必须覆盖完整性、一致性、可行性、可验证性、追溯和领域约束。
- 系统架构评审必须覆盖需求分配、接口、失效/异常行为、关键风险、资源和集成策略。
- G2/G3 由系统负责人组织，产品、软件、硬件、测试、质量及适用领域负责人共同评审。

## 10. 配置与变更控制

系统需求、架构、接口、外部分配清单、决策、追溯和评审记录均为配置项。系统基线变更必须分析软件、`EXT-HW`、测试、安全、网络安全、项目和发布影响，并关联批准及验证记录。

## 11. 度量指标

| 指标 | 数据来源 |
|---|---|
| 产品需求到系统需求覆盖率 | 系统追溯矩阵 |
| 系统需求架构分配率 | 系统追溯矩阵 |
| 系统需求可验证率 | 需求评审记录 |
| 接口问题关闭率 | 接口/评审问题台账 |
| 系统基线变更率 | 变更记录 |

## 12. 裁剪规则

- 小型纯软件产品可合并系统上下文、系统需求和系统架构说明，但必须保留边界、外部接口、需求分配、验证准则和追溯。
- 没有硬件或外部组件分配时，可将 `EXT-HW` 标记为不适用并记录理由；存在分配时不得删除外部规格、接口和集成验证关系。
- 跨团队接口、客户接口、安全或网络安全相关系统不得裁剪系统架构和接口评审。

## 13. 实施证据

- 系统上下文、系统需求规格和评审记录。
- 系统架构、接口控制说明和架构决策记录。
- 系统需求/设计基线及批准记录。
- 系统追溯矩阵、`EXT-HW` 外部分配清单、集成策略和问题关闭证据。
